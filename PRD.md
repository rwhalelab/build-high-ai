# Build-High 제품 요구사항 정의서 (PRD)

# 1. 프로젝트 개요

서비스 명칭: Build-High (빌드하이) 
핵심 가치: 엔지니어를 위한 스터디 및 프로젝트 팀 빌딩 매칭 플랫폼 
기술 스택: Next.js (App Router), Supabase, Tailwind CSS, v0.dev 
타겟 유저: 대학생 개발자, 취준생, 사이드 프로젝트 희망자 
디자인 컨셉: v0 기반의 전문적이고 깔끔한 UI 

# 2. 단계별 구현 로드맵 

Phase 1: 수업 내 달성 (3일 MVP) 

목표: 3일 안에 서비스의 엔드 투 엔드(End-to-End) 흐름을 완성하여 실제 유저가 가치를 느낄 수 있도록 구현
인증 (Auth): Supabase Auth 기반의 구글 로그인 기능을 통해 즉시 유저를 식별 
기초 프로필: 로그인한 유저가 자신의 '기술 스택(배열)'을 등록하고 수정할 수 있는 마이페이지 제공 
지능형 CRUD:
작성: 제목, 카테고리, 본문을 입력하면 AI가 자동으로 요약과 태그를 생성하여 저장 
조회: 메인 페이지에서 AI가 생성한 요약문이 포함된 카드 리스트를 확인하고 상세 페이지로 이동 
삭제: 본인이 작성한 글만 삭제 가능 (데이터 정합성 유지) 
AI 로직 (Sync-Save): 사용자 글 저장 시점에 AI API(Gemini 등)를 호출하여 **요약문(3줄)**과 **핵심 기술 태그(5개)**를 추출해 DB에 직접 기록 
표준 UI: v0.dev

Phase 2: 수업 이후 (개별 고도화) 
데이터 시각화: Recharts 등을 사용하여 플랫폼 내 기술 트렌드 및 유저 성장을 차트로 구현 
미디어 관리: 프로젝트 소개를 위한 이미지 업로드 및 파일 공유 기능 추가 
실시간 상호작용: 유저 간 1:1 채팅, 지원 알림 기능 지원 
고도화된 필터링: 유저 프로필과 게시글 스택을 비교하는 매칭 알고리즘 도입 
AI 실시간 어시스턴트: 글 작성 중 '스트리밍' 방식으로 AI 실시간 피드백을 주는 고도화된 에디터 제공 

# 3. 데이터 아키텍처 (DB 스키마) 

사용자 정보 그룹 (profiles) 
역할: 시스템의 근간으로, 구글 로그인 시 부여되는 고유 식별자(UUID)를 기준으로 유저 정보 보관 
주요 컬럼: 유저 이름, 프로필 사진, '보유 기술' 데이터(단순 텍스트 배열) 

지능형 게시글 그룹 (posts / studies) 
역할: 작성자의 원문과 AI가 가공한 데이터를 함께 저장 
주요 컬럼:
id: uuid (Primary Key) 
title: varchar(100), NOT NULL 
category: text (Development, Study, Project 중 선택) 
summary: AI가 생성한 3줄 요약 (리스트 카드 노출용) 
content: 상세 페이지용 본문 내용 
tags: AI가 추출한 핵심 기술 태그 배열 (jsonb) 
contact: 외부 연락처 링크 
created_at: timestamptz (Default: now()) 

플랫폼 통계 그룹 (stats) 
역할: 별도의 테이블 없이 전체 게시글 수나 유저 수를 실시간으로 계산(Aggregate)하여 대시보드에 노출 

# 4. UI/UX 구조 및 v0.dev 가이드 

메인 탐색 페이지 (Discovery): 
상단에 서비스 규모를 보여주는 3개의 스탯 카드(총 게시글, 유저 수 등) 배치 
리스트 카드는 제목보다 AI 요약문을 상단에 배치하여 가독성 향상 

지능형 작성 에디터 (Creation): 
작성 중 우측 'AI 가이드' 영역이 반응하며, 완료 버튼 클릭 시 AI 분석 결과를 확인하는 팝업/사이드바 구현 

상세 정보 페이지 (Deep Dive): 
본문과 함께 AI가 추출한 기술 태그를 뱃지(Badge) 형태로 강조 

# 5. AI 연동 핵심 시나리오 

시나리오: 데이터 전처리 (Pre-processing) 
동작: 유저가 게시글을 [저장]하면 DB 저장 전 AI 검문소 통과 
과정: AI가 본문을 읽고 기술 스택 판단 후 태그 및 요약 생성 
결과: 가공된 고품질 데이터 저장으로 검색 성능 및 유저 경험 상승 

# 6. 품질 관리 (QA) 및 보안 

빈 상태 (Empty State): 리스트 부재 시 "첫 번째 프로젝트의 주인공이 되어보세요!" 안내 출력 
유효성 검사: 제목 최소 5자 이상, 필수 항목 누락 시 등록 차단 
보안: Supabase의 **RLS(Row Level Security)**를 설정하여 데이터 무단 수정 방지
